# 自测用例表

> Belongs to: `05_context/writing-standards/`  
> Source: PRD 需求自查清单 V3.3 · Sheet 07  
> Version: v1.0.0

适用场景：研发交付前自测、PM 冒烟检查、AI 实现结果自查。

用于定义最小可跑路径：主流程能否跑通、关键异常能否拦住、权限边界是否穿透、数据结果是否符合口径。

关键提醒：验收标准表定义"做到什么算完成"；自测用例表定义"交付前怎么跑一遍确认没偏"。

## 空白模板

```
用例编号 | 用例类型 | 前置条件 | 操作步骤 | 测试数据 | 预期结果 | 覆盖规则/异常 | 是否通过 | 备注
```

| 列名 | 含义 |
|------|------|
| 用例编号 | 唯一标识，如 TC001 |
| 用例类型 | 主流程 / 校验用例 / 重复用例 / 权限用例 / 异常用例 / 数据用例 |
| 前置条件 | 执行用例前必须满足的条件 |
| 操作步骤 | 用例的操作路径 |
| 测试数据 | 使用的具体测试数据 |
| 预期结果 | 正确的期望结果 |
| 覆盖规则/异常 | 对应的规则编号（R001）或异常类型（空态） |
| 是否通过 | 执行后填写：通过/失败 |
| 备注 | 冒烟必跑标记、特殊说明 |

## 示例

| 用例编号 | 用例类型 | 前置条件 | 操作步骤 | 测试数据 | 预期结果 | 覆盖规则/异常 | 是否通过 | 备注 |
|---------|---------|---------|---------|---------|---------|-------------|---------|------|
| TC001 | 主流程 | 用户有新增权限 | 进入列表 > 点击新增 > 填写字段 > 保存 | 模板名称：审批提醒模板 | 保存成功，列表展示新增数据 | R001 / A001 | 待填写 | 冒烟必跑 |
| TC002 | 校验用例 | 模板名称为空 | 点击新增 > 不填模板名称 > 保存 | 模板名称为空 | 阻断保存，提示请输入模板名称 | V001 | 待填写 | |
| TC003 | 重复用例 | 已存在同名模板 | 新增同名模板并保存 | 模板名称：审批提醒模板 | 阻断保存，提示名称重复 | V002 | 待填写 | |
| TC004 | 权限用例 | 普通用户登录 | 进入模板列表 | 普通用户账号 | 不展示新增、编辑、删除按钮 | 权限规则 | 待填写 | |

## 注意事项

- 自测用例不是全量测试用例——是最小集合，用于快速验证主流程未断裂。
- 每个验收标准 A001/A002... 应该至少有一个对应的自测用例。
- L2 及以上需求必须包含：主流程 x1、校验用例 x1、权限用例 x1（如有角色差异）。
