# 验收标准表

> Belongs to: `05_context/writing-standards/`  
> Source: PRD 需求自查清单 V3.3 · Sheet 07  
> Version: v1.0.0

适用场景：需求完成判断、测试验收、业务验收、上线验收。

验收标准必须可观察、可测试、可判断。不能写"功能正常即可"。

## 空白模板

```
验收编号 | 验收对象 | 前置条件 | 操作/场景 | 期望结果 | 验收方式 | 优先级 | 通过标准 | 备注
```

| 列名 | 含义 |
|------|------|
| 验收编号 | 唯一标识，如 A001 |
| 验收对象 | 被验收的功能或场景 |
| 前置条件 | 执行验收前必须满足的条件（如"用户具备模板管理权限"） |
| 操作/场景 | 验收时执行的操作或模拟的场景 |
| 期望结果 | 正确的结果是什么 |
| 验收方式 | 功能测试 / 权限测试 / 数据验证 / 性能测试 |
| 优先级 | P0/P1/P2——P0 验收项是上线必过项 |
| 通过标准 | 判定通过的客观条件（不能写"功能正常即可"） |
| 备注 | 关联的规则编号、对应用例编号 |

## 示例

| 验收编号 | 验收对象 | 前置条件 | 操作/场景 | 期望结果 | 验收方式 | 优先级 | 通过标准 | 备注 |
|---------|---------|---------|---------|---------|---------|--------|---------|------|
| A001 | 新增模板 | 用户具备模板管理权限 | 填写必填字段后点击保存 | 模板保存成功，列表新增一条记录 | 功能测试 | P0 | 保存成功，字段完整，状态正确 | 同步校验更新时间 |
| A002 | 重复校验 | 已存在同名模板 | 再次新增同名模板并保存 | 系统阻断保存并提示重复 | 功能测试 | P0 | 不产生重复数据 | 按同业务模块查重 |
| A003 | 权限控制 | 普通用户无编辑权限 | 普通用户进入模板列表 | 不展示编辑按钮 | 权限测试 | P0 | 无权限用户不可编辑 | 需覆盖接口权限 |
| A004 | 空态展示 | 当前无模板数据 | 进入模板列表页 | 展示空态提示 | 功能测试 | P1 | 页面无报错，空态文案正确 | |

## 不合格写法

| 问题写法 | 为什么不合格 |
|---------|-------------|
| 功能正常即可 | 无法判断什么算正常 |
| 符合需求即可 | 没有可观察的判定标准 |
| 用户体验良好 | 无法测试，无法量化 |
