# Run Log Template

> Belongs to: `04_templates/`
> Purpose: 跨节点的持续性运行日志 — 修订根因、痛点证据、运行时间线、用户指正、复盘触发状态
> Governed by: `01_workflow/workflow-protocol.md` §Run Log Checkpoints
> Authority: `04_templates/run-log.schema.json`
> Version: v2.0.0

所有时间字段统一使用任务所在时区，精确到秒，格式为 `YYYY-MM-DD HH:mm:ss`。不得只写日期或只精确到分钟；自动记录器使用运行环境本地时区，手工记录必须沿用同一口径。

---

## 为什么需要 Run Log

PRD workflow 的主节点和 Node 4.5 分布在多次对话中。复盘时 Agent 只能靠对话记忆推导“为什么初版漏了操作流程”——但对话记忆随上下文长度衰减。Run Log 是这些证据的持久载体。

它不替代任何现有文件：
- `02-决策账本.md` 是全流程 append-only 决策账本，Run Log 记录过程证据并通过决策编号引用它
- `07-修订记录.md` 记录"改了什么"，Run Log 记录"**为什么漏了、第几轮发现的、根因是什么**"
- `08-Skill复盘沉淀建议.md` 是达到 Manifest 条件时生成的复盘产物，Run Log 是**生成它所需的首要证据**

## 写入时机

| 时机 | 写入字段 | 触发规则 |
|------|---------|---------|
| 每次节点开始 | 运行时间线 | 新节点启动时追加一行 |
| 每次节点完成 | Node 完成记录 | 更新该节点的结束时间和统计 |
| 对齐后出现新决策 | 决策账本 | 追加到 `02-决策账本.md`，同时在运行时间线中记录 |
| 每次修订完成 | 修订记录 | 即时追加，必须填"根因分类"和"checklist 模块" |
| 发现"初版不该漏"的问题 | 痛点日志 | 即时追加，必须填"初版为什么漏了" |
| 用户指正 Agent 理解/结构/规则 | 用户指正记录 | 即时追加，记录原话和指正类型 |
| 复盘触发状态变化 | 复盘触发状态 | 同类根因出现 ≥2 次或等级升级时立即更新 |

## 复盘消费规则

复盘 Agent 启动时：

1. 先读 Run Log 的"修订记录"和"痛点日志"作为首要证据
2. 统计"根因分类"分布：缺方法 X 次 / 缺模板 Y 次 / 缺门禁 Z 次
3. 同类根因 ≥ 2 次 → 必须出 patch
4. 同类根因 ≥ 3 次 → patch 优先级升至 P0
5. 偶发（1 次）→ 仅记录，不出 patch

复盘完成后，在 Run Log 末尾追加：

```markdown
## 复盘消费记录

| 时间 | 消费的痛点 | 产出的 patch | 是否已写入 |
|------|-----------|-------------|-----------|
| | | | 是 / 否 |
```

## 禁止行为

- 不得在 Run Log 中重复维护完整决策内容 — 决策归 02，过程证据归 09
- 不得跳过"根因分类"字段 — 没有分类的修订记录对复盘无用
- 不得在复盘消费前删除 Run Log
- 不得等"整个 PRD 做完"再一次性补写 Run Log — 它是实时日志，不是事后回忆
