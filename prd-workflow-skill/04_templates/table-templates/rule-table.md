# 规则表

> Belongs to: `05_context/writing-standards/`  
> Source: PRD 需求自查清单 V3.3 · Sheet 07  
> Version: v1.0.0

适用场景：业务规则、操作规则、默认规则、状态规则。

必须回答：什么条件下，谁对什么对象做什么，系统如何处理，处理后结果是什么。

## 空白模板

```
规则编号 | 规则对象 | 适用场景 | 触发条件 | 规则说明 | 系统处理 | 处理后结果 | 异常处理 | 适用角色 | 优先级 | 备注
```

| 列名 | 含义 |
|------|------|
| 规则编号 | 唯一标识，如 R001 |
| 规则对象 | 这条规则约束的业务对象（如"审批提交""删除操作"） |
| 适用场景 | 规则在哪个业务流程或页面中生效 |
| 触发条件 | 什么情况下触发这条规则（如"申请人点击提交审批"） |
| 规则说明 | 规则内容——什么条件下允许/禁止/触发 |
| 系统处理 | 系统执行什么操作 |
| 处理后结果 | 系统处理后数据/状态变成什么 |
| 异常处理 | 规则不满足时怎么处理（如"校验失败时阻断提交并提示原因"） |
| 适用角色 | 这条规则对谁生效 |
| 优先级 | P0/P1/P2 |
| 备注 | 关联的状态机、校验规则或其他依赖 |

## 示例

| 规则编号 | 规则对象 | 适用场景 | 触发条件 | 规则说明 | 系统处理 | 处理后结果 | 异常处理 | 适用角色 | 优先级 | 备注 |
|---------|---------|---------|---------|---------|---------|-----------|---------|---------|--------|------|
| R001 | 审批提交 | 作业票审批 | 申请人点击提交审批 | 仅当必填字段完整且当前状态为草稿时允许提交 | 校验字段完整性并生成审批任务 | 作业票状态变为待审批 | 校验失败时阻断提交并提示原因 | 申请人 | P0 | 需同步状态机 |
| R002 | 删除操作 | 模板管理 | 管理员点击删除 | 已被引用的数据不允许删除 | 判断引用关系 | 未引用则删除，已引用则阻断 | 提示"该数据已被引用，无法删除" | 管理员 | P1 | 建议逻辑删除 |

## 不合格写法

| 问题写法 | 为什么不合格 |
|---------|-------------|
| 支持审批 | 没写触发条件、处理规则、处理结果 |
| 管理员可以删除 | 没写哪些状态可删、被引用后能否删除 |
| 默认展示最新数据 | 没写"最新"的排序字段和时间口径 |
