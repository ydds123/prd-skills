# 商业分析深度检查

> Supporting reference for the `check-prd` skill. 在执行商业分析深度检查时加载；单独使用时请先提供待检查的 PRD 内容。

## 维度说明

> **适用范围：本维度仅适用于商业化产品（对外销售/运营的产品）。自研内部系统不适用，跳过此维度，改为在10.3节做ROI分析即可。**

B端产品最终要为企业创造商业价值。《决胜B端》强调产品经理必须"定义问题、解决问题、为价值交付负责"，商业分析是确保产品方向正确的关键。特别是SaaS产品，商业模式的可行性直接决定产品的生死。

**核心理论框架：**

1. **产业链分析框架**（《决胜B端》3.1节）：从宏观的产业链视角理解业务大盘——行业上下游关系、价值链分布、各环节的利润构成和核心玩家。

2. **市场结构分析模型**（《决胜B端》3.1节）：金字塔形、枣核形、倒三角形、哑铃形四种市场结构，决定了SaaS产品的可行性和定价策略。

3. **SaaS商业模式验证**（《决胜B端》第11-12章）：SaaS产品的核心指标包括CAC（客户获取成本）、LTV（客户生命周期价值）、MRR/ARR（月/年经常性收入）、Churn Rate（流失率）。LTV>3倍CAC是SaaS健康的基本门槛。

4. **PRD商业分析框架**（《决胜B端PRD模板》第3节）：目标市场与客户分析（目标市场、市场规模、市场特征、发展趋势、客户画像、客户痛点、卖点提炼）+ 竞品分析（商业模式、目标客户、运营推广策略、市场份额、产品分析、核心价值）。

5. **需求交付效率分析**（《决胜B端》第10章）：通过需求交付周期、PRD编写周期、研发等待周期等指标衡量产品团队效率和价值交付能力。

## 检查清单

### 10.1 市场与行业分析

- [ ] **产业链定位**
  - 检查内容：是否分析了产品在产业链中的位置和价值
  - 优质标准：绘制了目标行业的产业链图，标注了各环节的核心玩家和利润分布，明确了本产品切入的环节和价值创造点
  - 问题标志：未分析产业链，不清楚产品在行业生态中的位置
  - 参考依据：《决胜B端》3.1节产业链分析

- [ ] **市场规模估算**
  - 检查内容：是否估算了目标市场的规模（TAM/SAM/SOM）
  - 优质标准：用自上而下或自下而上的方法估算了市场规模（如"全国约有2000万个门店，按每家5000元/年计算，总市场规模约1000亿"），数据来源有引用
  - 问题标志：无市场规模分析，或数据凭空捏造
  - 参考依据：《决胜B端PRD模板》第3节——"市场规模：产品切入市场的规模"

- [ ] **市场特征与趋势**
  - 检查内容：是否分析了目标市场的竞争格局和发展趋势
  - 优质标准：描述了市场当前的竞争态势（如"两强竞争+多方蚕食"）、进入门槛、关键成功因素和发展趋势
  - 问题标志：市场描述过于乐观（如"蓝海市场""无竞争"），或缺乏具体竞争态势分析
  - 参考依据：《决胜B端PRD模板》第3节——市场特征、发展趋势

### 10.2 商业模式分析

- [ ] **收入模式清晰度**
  - 检查内容：是否明确了产品的收入模式
  - 优质标准：说明了产品如何赚钱（订阅制/按量计费/一次性许可/交易抽成/增值服务等），有基本的定价逻辑和价格区间参考
  - 问题标志：商业化产品但无收入模式说明；或自研系统但未分析成本收益比
  - 参考依据：《决胜B端》SaaS商业模式分析

- [ ] **客户获取策略**
  - 检查内容：是否说明了如何获取目标客户
  - 优质标准：描述了获客渠道（直销/渠道/线上/口碑）、获客成本预估、转化漏斗设计；对于SaaS产品，有免费试用/Freemium/PLG等获客策略的考量
  - 问题标志：只设计了产品功能，完全未考虑如何让客户知道并购买
  - 参考依据：《决胜B端PRD模板》第3节——竞品运营、推广、营销策略

- [ ] **SaaS关键指标预估**（SaaS产品必查）
  - 检查内容：是否分析了SaaS核心商业指标
  - 优质标准：预估了CAC（客户获取成本）、LTV（客户生命周期价值）、目标续约率/流失率、回本周期；LTV/CAC比值>3
  - 问题标志：SaaS产品未分析LTV/CAC，不清楚商业模式是否可持续
  - 参考依据：《决胜B端》SaaS商业模式——LTV>3xCAC是基本健康线

### 10.3 投入产出分析

- [ ] **项目投入估算**
  - 检查内容：是否估算了项目的研发投入（人力、时间、资金）
  - 优质标准：有粗略的工期估算和资源需求（如"预计需要5名研发，3个月完成V1.0"），标注了关键里程碑
  - 问题标志：完全没有投入估算，无法评估项目的可行性
  - 参考依据：《决胜B端》第10章项目管理——需求交付周期和工时利用率分析

- [ ] **收益预期与ROI**
  - 检查内容：是否量化了项目的预期收益，并计算了投入产出比
  - 优质标准：
    - 商业化产品：预估了收入增长或成本节约的具体金额
    - 自研系统：预估了效率提升幅度（如"预计减少50%的人工操作时间"）或风险降低程度
    - ROI=（收益-投入）/投入，有明确的回本预期
  - 问题标志：无收益预期，或收益预期无法量化
  - 参考依据：《决胜B端PRD模板》第4节——"具体的量化的项目目标"和"收益预期，需要量化"

- [ ] **机会成本考量**
  - 检查内容：是否考虑了做这个项目的机会成本——同样的资源投入其他项目是否更有价值
  - 优质标准：将本项目与其他备选项目对比，说明了为什么优先做本项目
  - 问题标志：项目决策缺乏优先级对比依据
  - 参考依据：B端产品资源管理最佳实践

### 10.4 数据驱动验证（商业化产品专项）

> 注：软件使用监控指标（用户/行为/业务/技术数据）适用于所有产品类型，已纳入 check-prd-14（维度14-运营方案与效果跟踪）的「上线跟踪与效果评估」中。本节仅检查商业化产品特有的售卖侧指标。

- [ ] **软件售卖分析指标**（仅适合商业化SaaS产品）
  - 检查内容：是否定义了衡量软件商业化情况的核心SaaS指标
  - 优质标准：覆盖以下两类数据：
    - **商业数据**：NDR（净收入留存率）、ARR（年度经常性收入）、MRR（月度经常性收入）、LTV（客户生命周期价值）、CAC（客户获取成本）；LTV/CAC > 3为健康基准线
    - **客户数据**：客户来源（自然流量/转介绍/采购流量），客户类型（付费/免费；活跃/流失/唤醒），客户级别（VIP客户/普通客户）
  - 问题标志：SaaS产品未关注NDR和LTV/CAC，不知道商业模式是否可持续；缺少客户来源分析无法优化获客
  - 参考依据：《决胜B端》SaaS商业模式分析；《决胜B端》产品监控的指标体系设计

- [ ] **数据采集方案**
  - 检查内容：是否设计了数据采集方案来支撑后续的数据分析
  - 优质标准：定义了需要埋点的核心页面和操作、数据采集频率、数据分析维度；与PRD第11节数据埋点章节呼应
  - 问题标志：定义了指标但不知道数据从哪来、如何采集
  - 参考依据：《决胜B端PRD模板》第11节数据埋点

## 常见问题诊断

| 问题现象 | 根本原因 | 改进方向 |
|---------|---------|---------|
| 无市场分析直接做功能 | 产品经理缺乏商业意识 | 先做市场和商业分析，再做功能设计 |
| 市场规模过于乐观 | 数据来源不可靠或计算逻辑有误 | 用可信数据源，自下而上交叉验证 |
| SaaS不算LTV/CAC | 不理解SaaS商业模式核心 | 补充SaaS核心指标分析 |
| 无投入估算 | 产品和技术沟通不充分 | 与技术负责人一起粗估工期和资源 |
| 收益无法量化 | 缺乏度量体系设计 | 定义可度量的成功指标，设计数据采集方案 |
