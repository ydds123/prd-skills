# 异常处理与健壮性设计检查

> Supporting reference for the `check-prd` skill. 在执行异常处理与健壮性设计检查时加载；单独使用时请先提供待检查的 PRD 内容。
> **注意**：运营方案与效果跟踪已独立为维度14，本维度专注于系统健壮性和风险管理。

## 维度说明

B端产品的真正考验不在于正常流程能否跑通，而在于异常情况能否妥善处理。《决胜B端》明确指出，产品方案"必须思考全面，包括异常情况的处理"。数据安全和合规性是B端产品不可忽视的底线，而提前识别并管理风险能让产品少走弯路。

**核心理论框架：**

1. **异常处理方案**（《决胜B端PRD模板》10.3节）：明确要求覆盖"断网、断电、误操作、数据丢失等"异常场景，异常处理方案可以写在专门章节或分散在各功能描述中。

2. **业务规则五种类型**（《决胜B端PRD模板》10.2节）：事实、约束、触发条件、推论、计算——其中"约束"类型专门描述"对业务动作和用户操作限制的描述"，包含"不符合约束条件时的预期响应"。

3. **项目风险管理**（《决胜B端PRD模板》第7节）：识别假设、约束、产品风险、运营风险、技术风险，并制定应对方案。

## 检查清单

### 12.1 异常场景覆盖

- [ ] **技术异常处理**
  - 检查内容：是否考虑了常见的技术异常场景
  - 优质标准：覆盖以下异常场景并有处理方案：
    - 网络中断/弱网环境：离线缓存、自动重试、断点续传
    - 服务不可用：友好的错误页面、降级策略
    - 数据加载超时：超时提示、取消/重试选项
    - 并发冲突：乐观锁/悲观锁策略、冲突提示
    - 数据丢失/损坏：自动备份、数据恢复方案
  - 问题标志：完全未考虑技术异常，默认系统永远正常运行
  - 参考依据：《决胜B端PRD模板》10.3节——"断网、断电"等异常

- [ ] **业务异常处理**
  - 检查内容：是否考虑了业务层面的异常场景
  - 优质标准：覆盖以下异常场景：
    - 误操作处理：支持撤销、回退、或管理员干预修正
    - 数据录入错误：校验前置、修改记录、审计追踪
    - 业务规则冲突：冲突检测、提示、人工裁决
    - 超时未处理：自动提醒、超时升级、自动取消/关闭
    - 角色缺失：审批人不在线/离职时的替代处理（代理审批、升级审批）
  - 问题标志：只考虑了理想业务路径，未考虑操作失误和业务冲突
  - 参考依据：《决胜B端PRD模板》10.3节——"误操作、数据丢失等"

- [ ] **边界条件处理**
  - 检查内容：是否考虑了数据和操作的边界条件
  - 优质标准：覆盖以下边界条件：
    - 空数据状态：列表为空时的友好提示和引导
    - 大数据量：分页、懒加载、导出限制
    - 并发操作：同一数据被多人同时编辑时的处理
    - 跨时区/跨区域：时区显示、多语言、货币转换
    - 极端输入：超长文本、特殊字符、超大文件
  - 问题标志：未考虑边界情况，用户在特殊条件下可能遇到系统崩溃或数据错误
  - 参考依据：B端产品健壮性设计最佳实践

### 12.2 数据安全与审计

- [ ] **操作审计设计**
  - 检查内容：是否设计了关键操作的审计追踪
  - 优质标准：关键业务操作（创建、修改、删除、审批、导出）有操作日志记录，包含操作人、操作时间、操作内容（修改前后值）、操作IP；日志不可篡改
  - 问题标志：无操作审计设计，出问题时无法追溯是谁做了什么操作
  - 参考依据：B端产品合规性要求

- [ ] **数据备份与恢复**
  - 检查内容：是否说明了数据备份策略和灾难恢复方案
  - 优质标准：定义了备份频率（如每日全量+实时增量）、备份存储位置、数据恢复的RTO（恢复时间目标）和RPO（恢复点目标）
  - 问题标志：无备份策略，或依赖"开发自己想办法"
  - 参考依据：企业级产品数据安全要求

- [ ] **敏感数据处理**
  - 检查内容：是否识别并保护了敏感数据（个人信息、财务数据、商业机密等）
  - 优质标准：识别了敏感数据字段并定义了保护措施（脱敏显示、加密存储、访问控制、日志审计），符合相关法规要求
  - 问题标志：包含敏感数据但无保护措施（如手机号/身份证号明文展示）
  - 参考依据：数据安全法规和B端产品安全设计

### 12.3 风险识别与应对

- [ ] **产品风险识别**
  - 检查内容：是否识别了产品层面的风险并制定了应对方案
  - 优质标准：识别了需求理解偏差风险、设计方案缺陷风险、用户接受度风险等，每个风险有概率评估和应对策略
  - 问题标志：PRD中无风险分析章节
  - 参考依据：《决胜B端PRD模板》第7节——"产品风险、运营风险、技术风险及应对方案"

- [ ] **技术风险识别**
  - 检查内容：是否识别了技术层面的风险
  - 优质标准：识别了技术可行性风险（如某功能技术上难以实现）、性能瓶颈风险、第三方依赖风险、安全风险等
  - 问题标志：假设所有技术方案都可行，未做风险评估
  - 参考依据：《决胜B端PRD模板》第7节

- [ ] **待决事项管理**
  - 检查内容：是否记录和管理了所有待决事项
  - 优质标准：每个待决事项有编号、描述、负责人、预计决策日期；待决事项数量控制在合理范围（<10个）；关键待决事项有备选方案
  - 问题标志：明显有未决问题但未记录；待决事项过多说明PRD不成熟
  - 参考依据：《决胜B端PRD模板》第14节待决事项

## 常见问题诊断

| 问题现象 | 根本原因 | 改进方向 |
|---------|---------|---------|
| 只有Happy Path | 未做异常场景推演 | 对每个核心流程做"如果出错/如果超时/如果冲突"推演 |
| 无操作审计 | 缺乏合规意识 | 为关键操作添加审计日志设计 |
| 敏感数据明文展示 | 未识别和保护敏感数据 | 识别敏感字段，设计脱敏方案和访问控制 |
| 风险无人管理 | 缺少风险管理章节 | 补充风险识别和应对方案 |
| 上线后系统频繁出问题 | 边界条件和异常场景未覆盖 | 系统性检查所有边界条件和异常场景 |
