# AI功能设计质量检查

> Supporting reference for the `check-prd` skill. 在执行 AI 功能设计质量检查时加载；单独使用时请先提供待检查的 PRD 内容。
> **注意**：本维度仅在PRD涉及AI功能时进行评估。如果PRD不涉及AI功能，标记为"不适用"并跳过。

## 维度说明

随着大语言模型（LLM）的快速发展，越来越多的B端产品开始集成AI能力。《决胜体验设计》第9章系统性地阐述了AI时代B端产品的交互设计方法论，提出了"六脉神剑"框架来指导AI功能的设计。正确的AI功能设计能极大提升产品价值，错误的设计则可能导致用户困惑甚至业务风险。

**核心理论框架：**

1. **人机交互三个历史阶段**（《决胜体验设计》9.1节）：以机器为中心的设计（19世纪末-20世纪80年代）→ 以人为中心的设计（20世纪80年代-现在）→ 以结果为中心的设计（AI时代）。AI时代的核心转变是用户告诉AI想要什么，AI直接交付结果。

2. **确定性-容错率二维模型**（《决胜体验设计》9.2节）：以"确定性任务vs探索性任务"为横轴，"容错率高vs容错率低"为纵轴，形成四个象限，不同象限适用不同的AI交互模式。

3. **AI交互六脉神剑**（《决胜体验设计》9.2节）：
   - **第一剑：封装API**——将AI能力封装在传统GUI按钮后面，用户点击按钮触发AI，适用于确定性高、容错率高的场景（如一键生成摘要）
   - **第二剑：将CUI嵌入GUI**——在GUI界面中嵌入对话式交互，适用于容错率高的探索性任务（如AIGC生成图片、文案）
   - **第三剑：AI Agent侧边栏**——统一的智能体侧边栏，在任何页面可唤起，适用于需要跨页面、多步骤操作的场景
   - **第四剑：AI辅助填充**——AI自动填充表单、自动推荐选项，适用于确定性高但容错率低的场景
   - **第五剑：AI后台自动化**——AI在后台自动执行任务无需用户干预，适用于完全确定性的自动化场景
   - **第六剑：全CUI替代GUI**——纯对话式交互，用户完全通过自然语言操控系统，适用于AI编程等高度灵活的场景

4. **AI Agent概念**（《决胜体验设计》9.2.4节）：AI Agent是承载AI从思考到落地闭环的实体，各大SaaS厂商（Salesforce的Agentforce、Atlassian的Rovo、HubSpot的Breeze、Shopify的Sidekick）都采用了智能体侧边栏的设计模式。

5. **Copilot模式**（《决胜体验设计》引用）：AI作为用户的助手（副驾驶）存在，用户保持控制权，AI提供建议和辅助。

## 检查清单

### 13.1 AI功能定位

- [ ] **AI功能的价值主张**
  - 检查内容：AI功能要解决什么问题，为用户带来什么价值
  - 优质标准：明确说明AI功能的目标（如"帮助销售人员自动生成客户跟进建议，节省每日30分钟的分析时间"），而非为了"AI化"而加AI
  - 问题标志：AI功能缺乏明确的价值主张（如"我们也加个AI功能"），或AI功能与产品核心价值无关
  - 参考依据：《决胜体验设计》9.1节——AI的核心价值是"以结果为中心"，让用户直接获得结果

- [ ] **AI适用场景判断**
  - 检查内容：所选场景是否适合用AI来解决
  - 优质标准：使用确定性-容错率二维模型判断场景所在象限，并选择了匹配的交互模式；AI适合用于数据分析、内容生成、模式识别、自动化等场景
  - 问题标志：在不适合AI的场景使用AI（如用AI替代简单的下拉选择，或在容错率极低的场景用不可靠的AI做自动决策）
  - 参考依据：《决胜体验设计》9.2节确定性-容错率二维模型

### 13.2 AI交互设计

- [ ] **AI交互模式选择**
  - 检查内容：AI功能的交互模式是否与场景匹配
  - 优质标准：根据六脉神剑框架选择了正确的交互模式：
    - 确定性高+容错率高→封装API（一键按钮）
    - 探索性+容错率高→CUI嵌入GUI（对话框）
    - 跨页面多步骤→AI Agent侧边栏
    - 确定性高+容错率低→AI辅助填充（需人工确认）
    - 完全自动化→AI后台自动化
  - 问题标志：交互模式与场景不匹配（如用全CUI替代简单的一键操作，或在高风险场景用无确认的自动化）
  - 参考依据：《决胜体验设计》9.2节AI交互六脉神剑

- [ ] **人机协作边界**
  - 检查内容：AI和人之间的职责边界是否清晰
  - 优质标准：明确了哪些步骤由AI完成、哪些步骤需要人工确认、哪些步骤人工不可绕过；在关键决策点保留了人工干预的能力（Human-in-the-Loop）
  - 问题标志：AI自动做出关键业务决策但用户无法干预或修改；或AI功能需要过多人工干预失去了效率优势
  - 参考依据：《决胜体验设计》Copilot模式——AI作为助手，用户保持控制权

- [ ] **AI输出的可解释性**
  - 检查内容：AI的输出结果是否对用户有解释性
  - 优质标准：AI给出建议或结果时附带了推理依据（如"基于该客户近30天的浏览记录和购买历史，推荐推送XX产品"）；用户能理解AI为什么给出这个结果
  - 问题标志：AI给出结果但不解释原因（"黑盒"输出），用户无法判断结果是否可信
  - 参考依据：AI产品可信赖设计原则

### 13.3 AI可靠性设计

- [ ] **AI错误处理与回退**
  - 检查内容：当AI输出错误或不可用时，是否有回退方案
  - 优质标准：设计了AI不可用时的降级方案（如AI推荐不可用时回退到规则引擎或手动操作）；AI生成内容有人工修改和覆盖的能力；AI服务异常时用户仍能完成核心操作
  - 问题标志：核心业务流程完全依赖AI，AI不可用时业务中断
  - 参考依据：B端产品可靠性设计原则

- [ ] **AI输出质量保障**
  - 检查内容：是否有机制保障AI输出的质量
  - 优质标准：定义了AI输出的质量标准和评估方法；有异常输出的检测和过滤机制；用户可以对AI输出进行反馈（点赞/点踩）以持续优化
  - 问题标志：对AI输出质量无任何管控，直接展示给用户或执行
  - 参考依据：AI产品质量管理

- [ ] **数据隐私与安全**
  - 检查内容：AI功能是否考虑了数据隐私和安全
  - 优质标准：说明了AI处理的数据范围和类型；明确了数据是否传输到外部AI服务（如调用第三方LLM API）；敏感数据（客户信息、财务数据）是否做了脱敏处理后再传给AI
  - 问题标志：将敏感业务数据直接传给第三方AI服务而无脱敏和安全措施
  - 参考依据：数据安全和隐私保护要求

### 13.4 AI体验优化

- [ ] **AI响应时间与加载体验**
  - 检查内容：是否考虑了AI处理的响应时间和等待体验
  - 优质标准：对于耗时较长的AI操作（如生成长文档），设计了流式输出（typing效果）、进度指示、异步通知等交互方式；用户在等待时可以继续其他操作
  - 问题标志：AI操作需要较长时间但无任何等待反馈，用户不知道系统在做什么
  - 参考依据：尼尔森第一原则——系统状态可见性

- [ ] **AI功能的渐进式呈现**
  - 检查内容：AI功能是否采用了渐进式设计，而非强迫用户使用
  - 优质标准：AI功能作为可选的辅助能力存在，不强制用户使用；新用户有引导了解AI能力；高级用户可以深度使用AI功能
  - 问题标志：AI功能强制替代了原有操作方式，不给用户选择权
  - 参考依据：《决胜体验设计》AI设计原则——渐进式引入AI能力

### 13.5 AI功能监控指标设计

AI产品的监控指标不同于普通功能，需要额外关注AI本身的能力表现。PRD中应定义上线后如何衡量AI功能是否真正有效。

- [ ] **使用效果指标**
  - 检查内容：是否定义了衡量AI能力表现的质量指标
  - 优质标准：覆盖以下维度：
    - **任务解决率**：问题解决率（用户采纳AI方案的比例）；监控方法：日志分析 + 行为分析 + 满意度评分
    - **理解准确性**：意图识别准确率、实体抽取准确率；监控方法：每日自动化测试 + 人工抽检
    - **回复质量**：答非所问率、回复完整度评分；监控方法：NLU模型评估 + 用户反馈
    - **多轮对话能力**（对话型AI适用）：平均对话轮次、上下文理解断裂率；监控方法：会话链路分析
  - 问题标志：只监控AI功能的使用量，不监控AI输出是否真的准确有效
  - 参考依据：《决胜体验设计》AI产品监控指标体系

- [ ] **用户行为指标**
  - 检查内容：是否定义了观察用户如何使用AI功能的行为指标
  - 优质标准：覆盖以下维度：
    - **使用深度**：提问复杂度分布、高级功能使用率；说明：引导用户加深使用
    - **放弃行为**：中途退出率、转人工率及原因分类；说明：定位AI能力覆盖不足之处
    - **问题类型**：高频问题TOP20、未覆盖问题占比；说明：指导知识库扩展方向
  - 问题标志：不分析放弃和转人工行为，无法发现AI能力瓶颈
  - 参考依据：《决胜体验设计》AI产品监控指标体系

- [ ] **推广渗透指标**
  - 检查内容：是否定义了衡量AI功能在团队/组织内渗透程度的指标
  - 优质标准：覆盖以下维度：
    - **覆盖率**：员工实际使用率（有多少人真的在用）
    - **替代率**：AI处理占比 vs 人工处理占比（AI实际替代了多少人工工作量）
    - **依赖度**：关键业务流程中AI参与度（AI是否真正融入日常工作）
  - 问题标志：只看功能上线没人问，不知道员工是否真的在用；无法回答"AI上线后到底替代了多少人工"
  - 参考依据：《决胜体验设计》AI产品监控指标体系

## 常见问题诊断

| 问题现象 | 根本原因 | 改进方向 |
|---------|---------|---------|
| 为AI而AI | 缺乏明确的AI价值主张 | 先明确AI要解决的具体问题和带来的具体价值 |
| AI交互方式与场景不匹配 | 未使用确定性-容错率模型判断 | 用六脉神剑框架选择正确的交互模式 |
| AI做了关键决策用户无法干预 | 人机边界设计不当 | 在关键决策点保留Human-in-the-Loop |
| AI不可用时业务中断 | 过度依赖AI缺乏降级方案 | 设计AI不可用时的回退方案 |
| 敏感数据泄露给第三方AI | 未做数据安全评估 | 对AI使用的数据做脱敏和安全审查 |
| AI输出不可信用户不敢用 | 缺乏可解释性设计 | 为AI输出添加推理依据和置信度指示 |
