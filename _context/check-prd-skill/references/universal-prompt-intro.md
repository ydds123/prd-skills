# B端PRD质量检查 Prompt

> 作者：杨堃，《决胜B端》作者，B端产品专家
> 基于：《决胜B端（第二版）》《决胜体验设计》《决胜B端PRD模板 v2.0》
> 本 Prompt 用于对 B端产品的 PRD 和系统设计方案做全面、系统化的质量检查，覆盖 14 个维度。

## 使用说明

1. 先把待检查的 PRD / 系统设计文档内容完整提供给模型，或上传文档
2. 再发送本 Prompt
3. 模型需要先完成产品定型，再按下述顺序逐维度检查，每个维度检查完立即输出详细分析，最后输出汇总报告

适用场景：

- PRD 评审前自查
- 设计方案质量回顾
- 新人 PRD 写作指导
- 产品方案完整性验证

## 第零步：产品定型

在开始任何维度检查前，先阅读文档并判断以下两个维度，记录结论供后续检查使用：

**A. 商业属性**

- 自研内部系统：企业内部使用，不对外销售，如内部 OA、ERP、CRM
- 商业化产品：对外销售或运营，存在付费客户或商业变现，如 SaaS、平台产品

**B. 功能类型**

- 业务型管理软件：支撑核心业务流程运转，有复杂业务对象和状态管理
- 工具型软件：提供独立功能提升效率，无复杂业务对象关系
- 交易型平台：撮合多方完成交易
- 基础服务型：提供底层能力供其他系统调用

**根据定型结论，以下维度按条件适用：**

| 维度 | 适用条件 | 不适用时处理方式 |
| --- | --- | --- |
| 07-数据建模（ER图） | 仅适用于业务型管理软件和交易型平台 | 工具型或基础服务型标注“不适用”，仅检查关键数据结构 |
| 10-商业分析 | 仅适用于商业化产品 | 自研内部系统仅执行投入产出分析 |
| 03 维度中的竞品分析 | 仅适用于商业化产品 | 自研内部系统改为同类系统参考 |
| 06-企业架构层 6.5-6.9 | 仅适用于 0-1 新系统设计或系统级规划 PRD | 迭代需求或模块级需求只做 6.1-6.4 |
| R8-多租户风险项 | 仅适用于商业化 SaaS 产品 | 非 SaaS 标注“不适用” |
| 13-AI 功能 | 仅在文档涉及 AI 功能时评估 | 否则标注“不适用” |

## 检查输出规范

**核心原则：逐维度即时输出。**

每完成一个维度检查，就立即输出该维度的详细分析，然后再继续下一个维度。禁止把所有维度攒到最后统一输出。

**每个维度使用以下格式：**

```md
## 维度[编号] - [名称] ｜ 评级：[优秀 / 合格 / 待改进 / 严重缺失]

### 具体发现

**发现 1：[问题标题]** [P0/P1/P2/P3]
- PRD定位：第X节/[功能名称]
- 问题描述：[具体说明问题，不要泛泛而谈]
- 改进示例：[给出可立刻执行的改法]

### 隐性问题推断
结合产品类型和业务场景，列出 PRD 没写但按道理必须考虑的问题。
```

**最低输出要求：**

- 每个维度至少输出 3 条具体发现，若确实无问题，也要逐条说明理由
- 每条发现必须引用 PRD 中的具体章节、流程、功能名或界面名
- 每条发现必须给出具体可执行的改进示例
- 维度 09 必须逐个检查文档中出现的界面、弹窗、表单、操作，并指出缺失的交互说明

## 检查执行顺序

### 第一阶段：业务与定位检查

1. 按下面的“维度01 - 业务分析质量”执行检查，并立即输出分析
2. 按下面的“维度02 - 产品类型适配性”执行检查，并立即输出分析
3. 按下面的“维度03 - 产品定位合理性”执行检查，并立即输出分析

### 第二阶段：场景与结构检查

4. 按下面的“维度04 - 场景分析与用户旅程”执行检查，并立即输出分析
5. 按下面的“维度05 - 文档结构完整性”执行检查，并立即输出分析
6. 按下面的“维度06 - 架构设计质量”执行检查，并立即输出分析

### 第三阶段：细节设计检查

7. 按下面的“维度07 - 数据建模质量”执行检查，并立即输出分析
8. 按下面的“维度08 - 流程与角色设计”执行检查，并立即输出分析
9. 按下面的“维度09 - 交互设计质量”执行检查，并立即输出分析

### 第四阶段：商业、演进与运营检查

10. 按下面的“维度10 - 商业分析深度”执行检查，并立即输出分析
11. 按下面的“维度11 - MVP策略与演进蓝图”执行检查，并立即输出分析
12. 按下面的“维度14 - 运营方案与效果跟踪”执行检查，并立即输出分析

### 第五阶段：健壮性与前瞻性检查

13. 按下面的“维度12 - 异常处理与健壮性设计”执行检查，并立即输出分析
14. 按下面的“维度13 - AI功能设计质量”执行检查，并立即输出分析

### 第六阶段：汇总报告

完成所有维度分析后，再按后面的“重大风险项清单”检查 R1-R8，并按“检查报告模板”输出汇总报告。
