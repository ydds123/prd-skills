# 重大风险项清单 + 常见问题对照表

> Supporting reference for the `check-prd` skill. 在所有维度检查完成后加载，用于识别重大风险项与高风险区域。

## 重大风险项清单

以下任何一项如果存在，建议在评审时重点讨论，推进前应有明确的解决方案。这些是产品设计的底线问题，影响系统的可用性或合规性。

### R1. 产品定位缺失
- **触发条件**：PRD通篇无法回答"这个产品是为谁解决什么问题"
- **判断标准**：阅读PRD后，无法用一句话概括产品定位
- **为什么重要**：没有定位，所有功能设计都没有方向
- **改进建议**：补充产品定位声明（为谁、解决什么问题、提供什么价值）
- **参考依据**：《决胜B端》第4章——产品定位是整体方案设计的核心

### R2. 核心业务流程缺失
- **触发条件**：PRD没有描述核心业务流程，或流程明显不完整
- **判断标准**：无法从PRD中还原出业务从开始到结束的完整路径
- **为什么重要**：不理解业务流程就无法设计正确的产品方案
- **改进建议**：用跨部门泳道图绘制核心业务流程
- **参考依据**：《决胜B端》2.1节——B端产品建设第一步是梳理核心业务流程

### R3. 数据模型严重缺陷
- **适用范围**：**仅适用于业务型管理软件和交易型平台**；工具型软件、基础服务型产品不适用此项
- **触发条件**：PRD涉及多个业务实体但完全无ER图，或ER图中实体关系明显错误
- **判断标准**：核心业务实体缺失、实体间关系错误（如一对多关系画成了多对多）、关键实体没有属性定义
- **为什么重要**：数据模型是系统的地基，一旦定型修改成本极高
- **改进建议**：重新进行ER建模，覆盖所有核心业务实体和关系
- **参考依据**：《决胜B端》5.1节——"ER建模最终会转换成数据库表设计，一旦表结构定型，以后再想修改，是非常麻烦的事情"

### R4. 角色权限完全缺失
- **触发条件**：PRD涉及多角色但完全没有权限设计
- **判断标准**：无法判断哪个角色能做什么、能看到什么
- **为什么重要**：B端产品的核心特征之一是"目标用户是一个群体，需要支撑分工协作"
- **改进建议**：建立角色权限矩阵，至少精确到页面级别
- **参考依据**：《决胜B端PRD模板》第12节角色权限矩阵

### R5. 功能存在致命断裂
- **触发条件**：核心业务流程中存在功能断点——某个环节无功能支撑导致业务无法继续
- **判断标准**：沿核心流程走查，发现"创建了但无法查询""提交了但无法审批""发货了但无法确认收货"等断点
- **为什么重要**：功能断裂意味着用户无法完成核心业务操作
- **改进建议**：沿核心流程逐一走查，补全所有断点
- **参考依据**：B端产品设计的业务闭环原则

### R6. 需求与业务严重脱节
- **触发条件**：PRD设计的功能明显不符合实际业务需求，或基于错误的业务假设
- **判断标准**：与业务方确认后发现PRD中描述的业务流程、角色关系、数据关系与实际不符
- **为什么重要**：基于错误假设的产品方案，做得越多错得越多
- **改进建议**：重新进行业务调研，校正业务理解
- **参考依据**：《决胜B端》第3章——业务调研是产品设计的前提

### R7. 合规性风险
- **适用范围**：涉及强监管行业（金融/医疗/教育/政务等）时适用
- **触发条件**：涉及强监管行业但完全未考虑合规要求
- **判断标准**：产品设计违反了已知的法规要求（如不做数据加密、不做实名认证、不做审计追踪）
- **为什么重要**：合规违规可能导致法律风险和企业损失
- **改进建议**：梳理行业合规要求，在PRD中补充合规设计
- **参考依据**：《决胜B端》调研计划中"政策法规"维度

### R8. SaaS产品无多租户概念
- **适用范围**：**仅适用于商业化SaaS产品**；企业自研内部系统不适用
- **触发条件**：商业化SaaS产品但完全无租户隔离概念
- **判断标准**：不同租户的数据可能互相可见，或功能配置无法按租户独立管理
- **为什么重要**：数据泄露是SaaS产品的生死线
- **改进建议**：补充多租户架构设计，明确数据隔离策略
- **参考依据**：《决胜B端》SaaS产品数据安全要求

## 常见问题对照表

以下汇总了13个维度检查中最高频出现的问题，按问题严重程度（P0-P3）排列，便于快速对照诊断。

### P0级问题（致命，必须修复）

| 编号 | 问题 | 所属维度 | 典型表现 | 修复建议 |
|------|------|---------|---------|---------|
| P0-01 | 产品定位缺失 | 03-定位 | 无法一句话说清产品是什么 | 明确"为谁+解决什么+提供什么价值" |
| P0-02 | 核心流程断裂 | 08-流程 | 业务无法完成闭环 | 走查核心流程补全断点 |
| P0-03 | ER模型关系错误 | 07-数据 | 一对多画成一对一 | 用业务场景逐一验证ER关系 |
| P0-04 | 权限完全缺失 | 08-流程 | 不知道谁能做什么 | 建立角色权限矩阵 |

|------|------|---------|---------|---------|

## 快速诊断流程

如果时间有限，可以按以下顺序进行快速诊断：

1. **先检查重大风险项**（R1-R8）——有触发的项明确告知，建议优先处理
2. **再检查P0级问题**——高优先级问题，建议尽快处理
3. **然后检查P1级问题**——建议上线前修订
4. **最后检查P2/P3**——作为改进建议输出

## 问题优先级说明

- **P0（高优先级）**：影响核心业务流程，或存在合规/安全风险，建议尽快处理
- **P1（建议修订）**：影响重要业务场景或用户体验，建议上线前解决
- **P2（可改进）**：影响次要场景或影响有限，按资源情况安排
- **P3（优化建议）**：能提升产品质量，有时间再做
